# i18n Implementation Plan: Indonesian, English, Chinese

## Overview

Add vue-i18n with URL-based locale prefixes (`/id`, `/en`, `/zh`) to the Verity+ blog portal. The app uses Vue 3 (TSX), Vue Router 5, Pinia, and Tailwind CSS.

---

## Architecture Decision

**URL-based locale routing** — Each locale gets its own URL prefix:

- `/id` — Indonesian (default/fallback)
- `/en` — English
- `/zh` — Chinese

A root `/` redirect will send users to their browser-detected locale or default to `/id`.

## Step 3: Create i18n Plugin (`src/i18n/index.ts`)

```ts
import { createI18n } from 'vue-i18n'
import { DEFAULT_LOCALE, SUPPORTED_LOCALES } from './types'
import id from './locales/id'
import en from './locales/en'
import zh from './locales/zh'

function detectLocale(): string {
  const path = window.location.pathname
  const segments = path.split('/').filter(Boolean)
  const firstSegment = segments[0]
  if (firstSegment && SUPPORTED_LOCALES.includes(firstSegment as any)) {
    return firstSegment
  }
  // Fallback: browser language or default
  const browserLang = navigator.language.split('-')[0]
  return SUPPORTED_LOCALES.includes(browserLang as any) ? browserLang : DEFAULT_LOCALE
}

export const i18n = createI18n({
  legacy: false,
  locale: detectLocale(),
  fallbackLocale: DEFAULT_LOCALE,
  messages: { id, en, zh },
})

export { SUPPORTED_LOCALES, DEFAULT_LOCALE }
export type { Locale } from './types'
```

## Step 4: Refactor Router for Locale Prefixes

Modify `src/router/index.ts`:

### Route Structure

Wrap all public-facing routes under a `/:locale` prefix. CMS routes stay at `/cms` (no locale needed for admin).

```ts
const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),
  routes: [
    // Locale-prefixed public routes
    {
      path: '/:locale(id|en|zh)',
      component: () => import('../features/pages/LocaleLayout'), // new wrapper
      children: [
        { path: '', name: 'home', component: () => import('../features/pages/HomeView') },
        { path: 'about-us', name: 'about-us', component: () => import('../features/pages/AboutUsView') },
        { path: 'contact', name: 'contact', component: () => import('../features/pages/ContactView') },
        { path: 'advertise', name: 'advertise', component: () => import('../features/pages/AdvertiseView') },
        { path: 'privacy-policy', name: 'privacy-policy', component: () => import('../features/pages/PrivacyPolicyView') },
        { path: 'terms-and-conditions', name: 'terms-and-conditions', component: () => import('../features/pages/TermsAndConditionsView') },
        { path: 'search', name: 'search', component: () => import('../features/search/views/SearchView') },
        { path: 'read/:slug', name: 'read', component: () => import('../features/article/views/ReadView') },
        { path: 'author/:id', name: 'author', component: () => import('../features/article/views/AuthorView') },
        { path: 'categories/:slug', name: 'category', component: () => import('../features/article/views/CategoryView') },
        { path: 'articles', name: 'all-articles', component: () => import('../features/article/views/AllArticlesView') },
      ],
    },
    // Root redirect — detect and redirect to /:locale
    {
      path: '/',
      redirect: () => {
        const detected = /* detect from navigator or cookie */ 'id'
        return `/${detected}`
      },
    },
    // CMS routes (unchanged, no locale prefix)
    { path: '/cms', component: () => import('../features/cms/views/CMSLayout'), children: [...] },
    // Catch-all 404
    { path: '/:pathMatch(.*)*', redirect: '/' },
  ],
})
```

### Navigation Guard Updates

- Add a guard that syncs vue-i18n locale from the `locale` route param
- Redirect unknown locales to `/id`
- On `router.push` calls, preserve the current locale

```ts
router.beforeEach((to, _from, next) => {
  // Sync i18n locale from route param
  if (to.params.locale && typeof to.params.locale === 'string') {
    i18n.global.locale.value = to.params.locale as Locale
  }

  // Root redirect
  if (to.path === '/') {
    const detected = detectLocale()
    return next(`/${detected}`)
  }

  next()
})
```

### Helper: `useLocaleRoute()`

Create a composable `src/composables/useLocaleRoute.ts` that wraps `useRouter().push` to automatically prepend the current locale:

```ts
export function useLocaleRoute() {
  const router = useRouter()
  const route = useRoute()
  const locale = computed(() => (route.params.locale as string) || DEFAULT_LOCALE)

  function push(to: RouteLocationRaw) {
    if (typeof to === 'string') {
      return router.push(`/${locale.value}${to}`)
    }
    // Handle object-based routes — inject locale param
    return router.push({ params: { locale: locale.value, ...to.params }, ...to })
  }

  return { push, locale }
}
```

## Step 7: Add Language Switcher Component

Create `src/components/ui/LanguageSwitcher.tsx`:

- Dropdown or button group showing ID / EN / ZH
- On click, navigates to the same route path but with the new locale prefix
- Place in Header (right side, next to SearchInput)

```tsx
// Switching from /id/about-us to /en/about-us
function switchLocale(newLocale: Locale) {
  const currentPath = route.path
  const pathWithoutLocale = currentPath.replace(/^\/(id|en|zh)/, '')
  router.push(`/${newLocale}${pathWithoutLocale}`)
}
```

## Step 11: Handle Edge Cases

1. **Invalid locale in URL** → Redirect to `/id`
2. **Root `/` access** → Redirect to browser-detected locale or `/id`
3. **CMS routes** → Keep at `/cms` without locale prefix (admin panel stays single-language for now)
4. **SEO** → Add `hreflang` meta tags for each locale variant
5. **PWA** — Ensure service worker handles locale-prefixed routes
6. **TypeScript** — Ensure full type safety on i18n keys
